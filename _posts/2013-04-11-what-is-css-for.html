--- title: "What is CSS for?" layout: post ---

HTML by itself is 100% responsive and 100% vector*. This is a simple but important point. And if it's correctly marked up with semantic tags, it's extremely readable on any device.

This is out of the box functionality that we should stop messing up. If your CSS removes any part of these features, you should rethink its purpose. If you solve a problem by creating one or more problems, you are doing a bad job.

When you rasterize the web,
when you make it less readable,
when you make it less responsive
- you create problems for your users.

Stop doing that.

CSS is not here to make our lives as developerseasy. It's here to enhance the usability of the web. It's here to make the users life better.

When we write CSS, we should keep these points in mind.
